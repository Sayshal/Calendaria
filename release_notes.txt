## New Features

- On PF2E and SF2E worlds, calendars without a Luxon sync theme now auto-align their displayed time-of-day with the system world clock's starting time, so the Calendaria clock matches the core Foundry clock without affecting calendar math
- Each calendar can now define its own Cross-Calendar display format in the editor's Display tab, controlling how its dates appear when shown as equivalents on other calendars
- New "Week Starts On" option in the calendar editor's Weeks tab rotates the mini and big calendar grids so a chosen weekday appears in the first column, without altering the underlying weekday cycle
- Short rests on dnd5e now advance world time independently of long rests, with a new Time-tab setting to choose Automatic (use the system's reported short rest duration) or Fixed minutes (default 60)

## Bug Fixes

- BigCal week view weekday labels now rotate with the Week Starts On setting so each column header matches its actual weekday
- Custom HUD format tokens with empty values (such as a missing canonical hour) now render as blank instead of leaking the literal token text
- Cycle conditions on notes now show the correct stage list and stop breaking when cycles or stages are reordered, with old saved values automatically still matching
- Players without Big Calendar access now see a permission warning toast instead of a momentary dialog flash when triggering the show-calendar macro
- Reordering months, weekdays, or seasons on older calendars no longer gets stuck swapping within the same group; existing calendars are migrated to stable random keys on load and Simple Calendar imports use them from the start
- Simple Calendar importer no longer misclassifies one-off dated notes as recurring festivals; festivals are only created for yearly-recurring or intercalary-anchored entries
- Simple Calendar imports now correctly remap month indices when intercalary months are stripped, so imported current dates and note dates land on the right Calendaria month instead of shifting
- The condition builder on a note now uses that note's calendar instead of always defaulting to the active one

## Changes

- "Week Starts On" setting moved from the Years tab to the Weeks tab in the calendar editor, with a clearer "First Weekday" label when left unset
- Fresh installs now boot with cinematic mode, weather/moon/event overlays, darkness and ambience scene syncing, HUD weather FX and border glow, chat timestamp clock, weather auto-generation, and FXMaster integration all turned off, so a new world stays quiet until you opt in; the in-HUD calendar widget is now shown by default
- The per-scene "Disable Weather FX" checkbox is now a "Weather FX Override" dropdown with Inherit, On, and Off, letting a scene force weather effects on even when the global setting is off, or off when the global is on; existing disabled scenes are migrated automatically
