## Video Walkthrough

A walkthrough of most major features, graciously created by the BaileyWiki team: https://youtu.be/yrl0Wh54ypQ

## New Features

- `[yearInEraOrdinal]` and `[yearInEraOrdinal=N]` format tokens for ordinal year-in-era display
- `advanceTime()` API now accepts object form `{ day, hour, minute, second }` and explicit `{ cinematic }` option
- Long rest time advancement for Pathfinder 1e
- Unsaved changes confirmation dialog in the Preset Manager

## Bug Fixes

- Chronicle entries loaded via infinite scroll not rendering rich text
- Chronicle not loading past entries on initial open
- Climate zone environment lighting and color shift settings not persisting
- D&D 5e long rests double-advancing time
- Explicit Edit actions ignoring Note Open Mode setting
- Intraday weather periods visible to non-GM players
- MiniCal close animation hanging
- Note permissions resetting on every save
- Notes Viewer FAB scrolling with page content
- Preset editor checkboxes always reading as false
- Preset reset not clearing color, icon, and owners
- Settings panel rendering on top of frameless Calendaria windows
- Stale ownership entries not cleaned up on visibility change
- Switching active climate zone not updating weather displays
- Sync to Notes not syncing ownership to journal documents
- Weather Editor name overrides not appearing in display paths
- Weather sounds persisting across scene and weather changes

## Changes

- D&D 5e long rests now trigger cinematic overlay when enabled (matching PF2e/PF1e)
- Enabling Intraday Weather regenerates all weather and forecasts
- Ownership controls on notes are GM-only
- Preset Manager reset consolidated to a single footer button
- Weather preset display names resolve: per-zone alias > Weather Editor override > built-in label
- Weather tooltip period breakdown requires GM or viewWeatherForecast permission
