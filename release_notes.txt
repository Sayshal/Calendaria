## Bug Fixes

- Cinematic not playing on long rest for individual actors or with Rest Recovery
- Climate editor losing preset weights when switching between seasons
- Cycle stages advancing every unit instead of respecting the configured length
- Darkness level flickering when advancing time by sub-hour increments
- Equinox and solstice festival notes anchoring to the wrong date
- Moon reference date producing wrong phases when yearZero is non-zero
- "Notify Before" set to 0 producing no notification
- Preset manager labels shifting between presets on select, reset, delete, and restore
- Random weather and manual weather selection ignoring the season temperature range
- Settings import failing between worlds due to world-specific user IDs
- Time inputs on note sheets displaying single-digit minutes (3 instead of 03)
- Weather sounds persisting when "Disable weather fx" is checked on a scene
- Weather system breaking at extreme temperatures
- Weekday reorder via Move Up/Down not persisting after save
- Old imported notes not affected by ownership sync on presets
- Zero-weight warning border not clearing after entering a value

## Changes

- `calendaria.ready` hook now fires before apps render; new `calendaria.rendered` hook fires after
- Default preset can now sync to all notes without a matching custom preset
- Season temperature range is now the authority for weather generation
- Settings import is resilient to individual failures instead of aborting entirely
- "Trigger on Rest" setting disabled when "Advance Time on Rest" is off
