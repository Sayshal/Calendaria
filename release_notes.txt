## New Features

- Cinematic overlay picks up the active theme so the date pill, progress bar, festival cards, and event highlights recolor to match themed worlds while default worlds stay on the warm-paper palette
- Sun, moon, and sky in the cinematic track the real in-world time between the skip's start and end hours, so opening and closing frames match the actual time of day

## Bug Fixes

- `[namedWeek]` and `[namedWeekAbbr]` format tokens resolve against the calendar's named weeks, honoring year-based vs month-based week type and the Repeat flag for cycling past the defined week count
- Cinematic pill fades in once the starting date is populated, so the first panel no longer flashes blank during the advance and keyframe-build window
- Computed seasonal notes and legacy seasonal migrations resolve astronomical anchors through season-type metadata, so recurring equinox/solstice triggers fire on the right day
- Custom format tokens that resolve to empty strings render the literal token instead of vanishing, matching the built-in token fallback behavior
- Equinox and solstice festivals land on the correct date for month-anchored calendars like Golarion and Pact-Standard, and for any calendar whose seasons are edited in the in-app season editor
- First cinematic frame seeds the date label immediately, and a malformed keyframe no longer crashes the animation loop
- Theme preset dropdown and Customize button unlock the moment Force Theme is set back to None, without requiring a settings reopen

## Changes

- Seasons left without a Seasonal Type are skipped by astronomical anchors; the season editor hint calls this out
